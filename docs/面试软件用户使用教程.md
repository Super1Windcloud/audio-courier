# Audio Courier 用户使用教程

## 1. 软件简介

Audio Courier 是一款桌面端实时语音转文字助手，支持：

- 实时语音转写
- 转写后继续向大模型发起对话
- 通过快捷键快速显示窗口、开始录音、发送消息
- 离线许可证激活
- 在线检查更新并安装新版本

适合会议记录、语音整理、采访草稿、面试问答辅助等场景。

## 2. 安装与启动

1. 双击安装包完成安装。
2. 首次启动后，如果系统弹出麦克风、网络或安全相关提示，请选择允许。
3. 软件启动后会显示主窗口。如果窗口被隐藏，可使用快捷键重新呼出。

## 3. 首次使用前要完成的 4 件事

首次打开软件，建议按下面顺序完成配置。

### 3.1 激活许可证

如果设备尚未授权，软件会自动提示激活。

操作步骤：

1. 点击顶部的`许可证`。
2. 在`用户标识`中填写你的用户编号，可不填。
3. 点击`生成请求码`。
4. 将生成的设备请求码发送给软件提供方。
5. 收到返回的 `license.json` 内容后，粘贴到`导入许可证`区域。
6. 点击`导入并激活`。

激活成功后，界面会显示`许可证已激活`。未激活时，发送消息和录音功能不可用。

### 3.2 配置转录 API

点击输入框右侧的`更多选项`，进入`转录 API`。

当前支持的转录供应商包括：

- Deepgram
- AssemblyAI
- Gladia
- RevAI
- Speechmatics

建议操作：

1. 选择你准备使用的供应商。
2. 填入对应的 API Key。
3. 如有需要，补充语言、模型或实时地址。
4. 点击`保存配置`。

说明：

- 这部分配置只影响语音转写。
- 默认中文可优先使用 `zh` 或 `cmn` 等语言代码，具体以供应商要求为准。
- macOS 用户如果需要转写系统输出音频，可在此处选择系统音频后端。

### 3.3 配置大模型 API

点击`更多选项` -> `模型 API`。

当前内置支持：

- SiliconFlow
- Doubao
- Kimi
- Zhipu
- DeepSeek
- Ali Qwen
- OpenAI
- Gemini
- 自定义 OpenAI 兼容接口

建议操作：

1. 先填好你要使用的模型供应商 API Key。
2. 如果使用 OpenAI、Gemini 或自定义兼容接口，可额外设置 Base URL 和模型名。
3. 点击`保存配置`。

说明：

- 这部分配置只影响大模型问答，不影响语音转写。
- 如果你的发行版已经内置可用额度，某些项目可能显示`免费额度`，此时可以先不填自己的 Key。

### 3.4 设置提示词

点击`更多选项` -> `提示词`。

这里有两类提示词：

- `常规对话提示词`：用于普通聊天与语音转写后的问答
- `自我介绍提示词`：仅用于自我介绍场景的首次请求

建议把长期角色设定、语言风格、回答要求写在这里，然后点击`保存提示词`。

## 4. 主界面说明

主界面主要由以下部分组成：

- 顶部左侧
  - `关于`：查看版本信息
  - `许可证`：查看授权状态或执行离线激活
- 输入区右侧按钮
  - `麦克风`：开始或停止语音转写
  - `清空`：清空当前对话
  - `发送`：发送当前文本
  - `更多选项`：打开配置菜单

## 5. 日常使用流程

### 5.1 选择转录厂商

点击`更多选项` -> `转录厂商`，选择当前要使用的语音转写服务。

### 5.2 选择大模型

点击`更多选项` -> `大模型`，选择当前问答所使用的模型。

### 5.3 选择音频通道

点击`更多选项` -> `选择音频通道`，选择要采集的声音来源。

可选来源通常包括：

- 麦克风输入
- 系统输出
- 默认音频设备

建议：

- 说话转写时，选择麦克风。
- 转写电脑播放内容时，选择输出设备或对应的系统音频通道。

### 5.4 开始语音转写

1. 点击麦克风按钮，或按下快捷键开始录音。
2. 软件会将识别中的文字实时写入输入框。
3. 识别稳定后的文字会持续累积到当前输入内容中。
4. 点击发送按钮，或按回车发送内容。

注意：

- 录音开始后，至少等待 3 秒才能停止。
- 未激活许可证时无法开始录音。

### 5.5 直接文本对话

如果你不需要录音，也可以直接：

1. 在输入框中键入文本。
2. 按 `Enter` 发送。
3. 使用 `Shift+Enter` 换行。

## 6. 更多选项说明

`更多选项` 菜单包含以下功能：

- `提示词`：编辑常规提示词和自我介绍提示词
- `模型 API`：配置大模型接口
- `转录 API`：配置语音转写接口
- `检查更新`：检测并安装新版本
- `前端透明度`：调整窗口透明度
- `文字颜色`：切换浅色文字或深色文字
- `快捷键`：查看软件内置快捷键
- `大模型`：切换当前问答模型
- `转录厂商`：切换当前转写供应商
- `选择音频通道`：切换录音来源
- `自动滚动到底部`：新消息自动滚动
- `捕获间隔`：调整音频捕获频率

## 7. 默认快捷键

当前版本内置快捷键如下：

- `Ctrl+Shift+\``：显示/隐藏窗口
- `Alt+Space`：切换录音
- `Enter`：发送消息
- `Shift+Enter`：输入框内换行；在全局快捷键说明中也用于快速发送当前消息
- `Ctrl+F12`：打开开发者工具

如果你忘记了快捷键，也可以在`更多选项` -> `快捷键`中查看。

## 8. 软件更新

软件支持在线检查更新。

操作步骤：

1. 点击`更多选项` -> `检查更新`。
2. 如果检测到新版本，会弹出更新窗口。
3. 点击`立即更新`。
4. 下载完成后，软件可能自动退出并启动安装器。

这是正常行为，不需要手动重复打开旧版本。

## 9. 常见问题

### 9.1 点击录音没有反应

请依次检查：

1. 是否已导入有效许可证。
2. 是否已配置至少一个可用的转录供应商 API Key。
3. 是否已选择正确的音频通道。
4. 系统是否已授予麦克风权限。

### 9.2 能录音，但没有转写结果

通常是以下原因：

- 转录供应商 API Key 无效
- 语言代码配置错误
- 当前音频通道选错
- 网络不可用或接口访问受限

建议先切换到一个你已确认可用的转录供应商后再测试。

### 9.3 可以转写，但模型不回复

请检查：

- 是否已配置大模型 API Key
- 当前选中的模型是否与你填写的供应商一致
- 自定义 Base URL 和模型名是否填写正确

### 9.4 导入许可证失败

请确认：

- 粘贴的是完整的 `license.json` 内容
- 当前许可证确实签发给这台设备
- 许可证未过期
- 当前软件版本没有超出许可证允许范围

### 9.5 窗口不见了

可先尝试按：

`Ctrl+Shift+\``

该快捷键可以重新显示或隐藏主窗口。

## 10. 建议的首次上手顺序

如果你是第一次接触本软件，建议直接按下面流程操作：

1. 安装并启动软件。
2. 完成许可证激活。
3. 在`转录 API`里填入一个可用的转录服务 Key。
4. 在`模型 API`里填入一个可用的大模型 Key。
5. 在`转录厂商`中选择对应供应商。
6. 在`大模型`中选择对应模型。
7. 在`选择音频通道`中选好麦克风或系统音频。
8. 点击麦克风开始说话。
9. 检查输入框里是否持续出现转写文本。
10. 点击发送，查看模型回复。

完成以上步骤后，即可正常使用。
