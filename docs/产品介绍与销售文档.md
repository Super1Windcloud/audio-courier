# Audio Courier 产品介绍与销售文档

## 一句话介绍

Audio Courier 是一款面向面试、会议、在线沟通和学习场景的本地桌面 AI 语音助手。它可以实时采集麦克风或系统音频，将语音快速转写成文字，并把转写内容交给大模型生成结构化回答、思路提示、总结或下一步建议。

## 产品定位

Audio Courier 不是普通录音软件，也不是单一的聊天机器人。它更像一个常驻桌面的“语音到智能回答”工作台：

- 前端负责轻量、透明、快捷的桌面交互。
- Tauri 原生能力负责音频采集、全局快捷键和本地配置。
- 多家转录供应商负责实时语音识别。
- 多家大模型供应商负责内容理解、回答生成和场景化辅助。

它适合需要边听、边转写、边获得 AI 辅助的人群，尤其适合远程面试、技术面试、销售沟通、在线会议、培训复盘和资料问答。

## 目标用户

- 求职者：在面试中快速理解问题，结合个人简历生成更清晰的回答思路。
- 面试辅导与职业服务团队：为学员提供实时问答辅助和标准化表达训练。
- 销售与客户成功团队：在客户沟通中实时整理需求、异议和应答建议。
- 会议密集型团队：把会议语音快速转成可追踪的文字和行动项。
- 内容创作者与学习者：把课程、播客、视频会议等音频内容转成可总结、可追问的文本。

## 用户痛点

1. 重要沟通中来不及记录  
   面试、客户会议和线上讨论通常信息密度高，手动记录会分散注意力，遗漏关键细节。

2. 听懂问题和组织回答之间存在时间差  
   很多场景不只是“听见”，还需要快速理解上下文、提炼重点，并组织成有逻辑的表达。

3. 单一 AI 工具不够灵活  
   不同用户可能使用不同的大模型或转录服务。固定供应商会限制成本、效果和区域可用性。

4. 浏览器工具不适合桌面实时场景  
   浏览器页面容易被遮挡、切换成本高，也难以稳定接入系统级音频和全局快捷键。

5. 面试类场景需要个人上下文  
   没有简历、项目经历和岗位目标作为背景，AI 回答容易泛泛而谈，缺少个人化说服力。

## 核心卖点

### 1. 实时语音转写，让沟通内容立即可用

Audio Courier 支持从音频通道采集声音，并通过 Deepgram、AssemblyAI、Gladia、Speechmatics、RevAI 等转录供应商进行实时识别。用户可以根据识别效果、语言支持、成本和网络环境切换供应商。

销售表达：

> 它把“听到的话”即时变成“可处理的文字”，让 AI 可以马上参与面试、会议和客户沟通，而不是等录音结束后再整理。

### 2. 多模型接入，避免被单一供应商绑定

产品内置 SiliconFlow、MiniMax、Doubao、Kimi、Zhipu、DeepSeek、通义千问、OpenAI、Gemini，并支持自定义 OpenAI 兼容接口。企业或个人可以按预算、效果和合规要求选择模型。

销售表达：

> 你不用为了一个工具迁移整个 AI 技术栈。已有 API Key 可以直接配置，私有化或第三方 OpenAI 兼容接口也能接入。

### 3. 面试场景友好，支持简历上下文和专用提示词

Audio Courier 支持配置常规对话提示词和自我介绍提示词，并可以导入 PDF / DOCX 简历作为上下文。对于求职者或面试辅导场景，AI 能基于个人经历生成更贴合自身背景的表达。

销售表达：

> 它不是给你一段模板答案，而是把你的简历、项目和岗位目标放进上下文，帮你组织更像“你自己”的回答。

### 4. 桌面级体验，适合高频即时操作

产品基于 Tauri 构建，支持桌面窗口、全局快捷键、显示/隐藏窗口、切换录音、快速发送等操作。透明度、文字颜色、自动滚动、音频通道和捕获间隔都可以在界面内调整。

销售表达：

> 面试或会议中不需要来回切窗口，用快捷键就能录音、发送和隐藏界面，减少操作打断。

### 5. 配置清晰，个人和团队都能落地

大模型 API 与转录 API 分开配置，互不影响。API Key 可以使用本地配置，也可以回退到环境预设，便于个人使用、团队分发和商业授权。

销售表达：

> 它既能给个人用户开箱使用，也能让团队统一配置模型、转录服务和默认提示词。

## 典型使用场景

### 场景一：远程面试实时辅助

用户提前导入简历，配置岗位相关提示词。面试开始后，Audio Courier 捕获面试官问题并转成文字，用户一键发送给大模型，快速获得回答结构、项目亮点和表达建议。

价值：

- 降低紧张导致的表达卡顿。
- 帮助用户围绕简历和项目经历回答。
- 提升回答的结构感和完整度。

### 场景二：客户会议与销售沟通

销售人员在会议中捕获客户语音，实时提炼客户需求、预算、异议和下一步动作。会后可基于转写内容生成跟进邮件或沟通纪要。

价值：

- 客户重点不遗漏。
- 异议处理更快。
- 跟进内容更准确。

### 场景三：在线会议记录与复盘

团队会议中，Audio Courier 可将讨论内容转写为文字，再通过大模型整理为摘要、决策、待办事项和风险点。

价值：

- 减少人工会议纪要成本。
- 让会议信息更容易检索和复盘。
- 提高跨团队同步效率。

### 场景四：课程、播客与视频学习

学习者可以捕获系统音频，把课程或视频内容实时转成文字，再让 AI 总结知识点、解释概念或生成复习提纲。

价值：

- 把被动听课变成可追问的学习过程。
- 快速沉淀笔记和复习材料。
- 提高长音频内容的吸收效率。

## 客户常见问题

### Q1：它和普通转写软件有什么不同？

普通转写软件主要负责把声音变成文字。Audio Courier 的重点是“转写之后马上处理”：它可以把实时转写内容发送给大模型，用于回答生成、总结、提炼和追问。

### Q2：它只能用于面试吗？

不是。面试是重点场景之一，但它同样适合会议纪要、销售沟通、在线课程、播客学习、资料问答和任何需要实时语音理解的场景。

### Q3：必须使用固定模型吗？

不需要。Audio Courier 支持多种内置模型供应商，也支持 OpenAI、Gemini 和自定义 OpenAI 兼容接口。用户可以按自己的账号和成本选择。

### Q4：转写供应商可以切换吗？

可以。当前支持 Deepgram、AssemblyAI、Gladia、Speechmatics、RevAI，并允许分别配置 API Key、语言、模型或实时地址等参数。

### Q5：能不能根据我的简历回答？

可以。产品支持导入 PDF / DOCX 简历文本，并将其加入提示词上下文中，帮助生成更贴合个人经历的回答。

### Q6：是否适合商业化分发？

适合。项目已经包含许可证状态校验、更新检查、Tauri 打包和供应商配置能力，可以作为个人产品、团队工具或垂直场景解决方案继续包装。
